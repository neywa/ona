ShiftFeed · Golden Gate

Beta Test Plan

Effective date: May 07, 2026  ·  Mobile only (Android + iOS)  ·  ~45 minutes

★ What you're testing

ShiftFeed is an OpenShift and Kubernetes news aggregator for SREs, DevOps engineers, and platform engineers. It pulls articles from Red Hat Blog, Kubernetes Blog, CNCF, HackerNoon, and Hacker News, tracks OCP stable releases, surfaces Red Hat CVEs, and generates an AI-powered daily briefing.

Scope of this round: the mobile app on Android and iOS only. Do not test the web build at neywa.github.io/shiftfeed — that is a separate target and is not in scope here.

The test plan is split into Free and Pro features. You can complete a full Free pass in about 30 minutes. Pro testing requires a promotional entitlement — see Section 3 for setup.

1 Environment setup

Android

  1. Install the Play Internal Testing build via the link the developer sent you.
  2. Minimum supported version: Android 7.0 (API 24). Ideally test on Android 13+ for push notification behaviour.
  3. Note the app version — you'll find it at the very bottom of the Settings screen.
  4. Confirm you have Wi-Fi or cellular data available. The app degrades gracefully offline but most features need a connection.

iOS

  1. Accept the TestFlight invite sent by the developer and install via TestFlight.
  2. Minimum supported version: iOS 13.
  3. Note the app version from the TestFlight app or from Settings inside ShiftFeed.
  4. Confirm network access as above.
Web and desktop builds are out of scope for this round. If you open neywa.github.io/shiftfeed in a browser, please do not file issues against it — that is a separate release track.

2 Pro tier setup

Skip this section if you're only testing Free features. Come back here when you reach a Pro-tagged test case.

  1. Open ShiftFeed → tap Settings (gear icon).
  2. Go to Account → Sign in → enter your email → tap "Send sign-in link".
  3. Open the magic-link email on the same device → tap the link → ShiftFeed opens and you're signed in.
  4. Back in Settings, scroll to the very bottom → tap Copy RevenueCat ID.
  5. Send that ID to the developer via email or Slack. They will grant a Pro promotional entitlement from the RevenueCat dashboard.
  6. Once confirmed, fully restart the app (swipe it away from recents and relaunch).
  7. A PRO badge should appear next to "ShiftFeed" in the top app bar.
  8. In Settings, the Notifications, Alert Rules, Daily Briefing, and Custom Feeds sections should show real content instead of upgrade prompts.
If the PRO badge doesn't appear after restarting, tap Settings → scroll to bottom → "Restore purchases" and try again.

3 How to report a bug

For every issue you find, please include:

  • Test ID — e.g. T-FEED-03
  • Device model and OS version — e.g. Samsung Galaxy S23 Ultra, Android 16
  • App version — from the bottom of the Settings screen
  • Steps to reproduce — what you did, in order
  • Expected vs actual — what should have happened vs what did
  • Screenshot or screen recording — if possible, attach one
✉
Send bug reports to: bobek.roman@gmail.com or via Slack as agreed with the developer.
